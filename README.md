### Установка и запуск

### Запуск локально
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Dobi-Vanish/L0
2. Перейдите в корень проекта и запустите через makefile поочерёдно команды:
   ```bash
   cd L0
   make build
   make docker-build
   make docker-up
   make migrate-up
3. Перезапустите контейнеры для применения всех настроек:
   ```bash
   make docker-down
   make docker-up

После успешного запуска доступны следующие endpoints:
 - Prometheus Metrics: `http://localhost:9090`
 - Статистика кэша: `http://localhost:8081/cache/stats`
 - Просмотр заказа по ID: `http://localhost:8081/`
 - Добавление заказа (через Postman): `http://localhost:8081/add_order`
Для подключения и просмотра логов через MongoDB подключиться через URI: `mongodb://localhost:27017`.

## Профилирование и оптимизация

### Инструменты
- **pprof** (endpoint `http://localhost:6060/debug/pprof/`) – CPU, heap.
- **go test -bench** – микробенчмарки и сравнение.
- **go tool trace** – анализ горутин.
- **hey** - для создания нагрузки и снятия бенчмарок во время нагрузки/без неё.

### Предпринятые действия
1. Создан тестовый заказ через `POST /add_order` с id test-order-1.
2. Нагрузка `hey -z 30s -q 1000 http://localhost:8081/order/test-order-1`.
3. Во время нагрузки снят CPU-профиль (`cpu_server_load.out`), heap-профиль (`heap_server_load.out`).
4. Обнаружено, что ~20% времени занимает `encoding/json` и сопутствующие аллокации (~170 на один заказ).
5. Проведены  оптимизации, повторены бенчмарки, результаты сравнены.

### Результаты
- Пропускная способность: ~35k req/s (hey, задержка p50=1.2ms).
- Количество аллокаций на заказ снизилось с **170 до 1**, что уменьшило нагрузку на GC.

Профили, файлы бенчмарков и скриншоты сохранены в папке `profiling/`.
