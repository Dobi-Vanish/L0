### Установка и запуск

### Запуск локально
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Dobi-Vanish/L0
2. Перейдите в корень проекта и запустите через makefile поочерёдно команды:
   ```bash
   cd L0
   make build
   make docker-build
   make docker-up
   make migrate-up
3. Перезапустите контейнеры для применения всех настроек:
   ```bash
   make docker-down
   make docker-up

После успешного запуска доступны следующие endpoints:
 - Prometheus Metrics: `http://localhost:9090`
 - Статистика кэша: `http://localhost:8081/cache/stats`
 - Просмотр заказа по ID: `http://localhost:8081/`
 - Добавление заказа (через Postman): `http://localhost:8081/add_order`
Для подключения и просмотра логов через MongoDB подключиться через URI: `mongodb://localhost:27017`.

## Профилирование производительности и история оптимизаций

Мы используем `net/http/pprof`, `go test -bench` и `go tool trace` для анализа и улучшения производительности сервиса.  
Результаты каждой оптимизации фиксируются в таблице ниже.

### Методология
- **Нагрузка**: `hey -z 30s -q 1000 http://localhost:8081/order/...`
- **Профили CPU/памяти**: снимаются через pprof на порту 6060/6061 во время нагрузки.
- **Бенчмарки**: `go test -bench=. -benchmem ./...`
- **Сравнение**: `benchstat old.txt new.txt`

### Текущие метрики производительности
- Пропускная способность API: ~12,000 запросов/сек на `GET /order/{uid}` (при кеш-попадании).
- Средняя задержка: <0.5 мс (p99) для кешированных ответов, <5 мс для запросов в БД.
- Потребление памяти потребителем Kafka: стабильно 50 МБ при 1000 заказах/сек.
